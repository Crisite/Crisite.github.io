存储过程

主要了解一下存储过程的概念、优缺点、怎么用，这方面就不深入了

存储过程

优点

  • 允许组件化编程，可以复用sql。（mybatis的xml也可以做到复用sql）
  • 较快的执行速度，存储过程是预编译的，在初次使用时会分析优化并存在系统的存储计划中。而T-sql语句每次执行都需要分析优化
  • 减少网络流量，因为存储过程是创建后保存在数据库的，因此我们客户端在调用的时候就不需要传递大量的sql语句，而是传递一个调用存储过程的指令+所需参数即可
  • 安全性，调用这只需要知道如何调用存储过程即可，而无需关心具体实现

缺点

  • 编写复杂，调试困难
  • 学习成本，存储过程过多会增加开发人员的学习成本
  • 依赖性，存储过程与特定的数据库系统关联，在数据迁移或的时候可能出现问题

SqlServer中使用存储过程

创建：

create proc | procedure 存储名(
  [{@参数 数据类型} [=默认值] [out|output],
   {@参数 数据类型} [=默认值] [out|output],
  ....]
)
as
begin
   SQL_statements
end

以创建有参存储过程为例

create procedure proc_name
(@var int=0)
as 
begin
  SELECT test_column from test_table where test_column=@var;
end
go

调用存储过程

EXEC 存储过程名称 参数
EXEC proc_name 2

SqlServer常用的系统存储过程

exec sp_databases; --查看数据库
exec sp_tables;        --查看表
exec sp_columns student;--查看列
exec sp_helpIndex student;--查看索引
exec sp_helpConstraint student;--约束
exec sp_stored_procedures;
exec sp_helptext 'sp_stored_procedures';--查看存储过程创建、定义语句
exec sp_rename student, stuInfo;--修改表、索引、列的名称
exec sp_renamedb myTempDB, myDB;--更改数据库名称
exec sp_defaultdb 'master', 'myDB';--更改登录名的默认数据库
exec sp_helpdb;--数据库帮助，查询数据库信息
exec sp_helpdb master;

Mysql中的存储过程

Mysql5.0以后才支持存储过程

-- 创建一个 myproc 存储过程
DELIMITER //           -- 把 // 定义为执行语法     （其实没有这个话也可以的）
  CREATE PROCEDURE myproc(OUT s int)
    BEGIN
      SELECT COUNT(*) INTO s FROM students;
    END
    //
DELIMITER ;

MySQL默认以”;”为分隔符，如果没有声明分割符，则编译器会把存储过程当成SQL语句进行处理，因此编译过程会报错，所以要事先用“DELIMITER //”声明当前段分隔符，让编译器把两个”//“之间的内容当做存储过程的代码，不会执行这些代码；“DELIMITER ;”的意为把分隔符还原。

调用：

SET @s=1;
CALL myproc(@s);
SELECT @S;
