Gateway简述

具体参数去官网查看文档即可

组成

spring cloud gateway 主要功能由三个部分实现:

  1. Router(路由)

一个 Route 模块由一个ID, 一个目标URI,一组断言和一组过滤器组成。如果断言为真，则路由匹配，目标URI会被访问。

  1. Predicate(断言)

可以用Predicate来匹配来自HTTP请求的任何内容。断言的输入类型是一个ServerWebExchange。

  1. Filter(过滤器)

在请求进入路由之前和之后对请求/相应进行各种处理，可以用于如身份验证，限流，转换等。过滤器为org.springframework.cloud.gateway.filter.GatewayFilter类的实例。

服务治理功能

spring:
  cloud:
    gateway:
      enabled: true
      routes: 
         #id是路由的唯一Id
       - id: noh-url
         # 实例是lb://服务名/，配置自己的新加项目的路径
         uri: lb://xxx/
         # 配置需要走路由的判断，此处为/xxx/xxx/**都走这个路由
         predicates: 
           - Path=/xxx/xxx/**
         # 路由的匹配顺序,新建的路由顺序需要在1000之内
         order: 200
        # 配置过滤器，这里是把/xxx/**转到具体的业务的时候，去掉/xxx/从而变为/**
         filters: 
           - StripPrefix=2

  1. 路由的请求与转发, 动态路由

每个route配置像都是一个路由, uri是转发请求的目标服务地址, 只有满足predicates中的断言条件的请求才会被转发. 动态路由通常是使用Java代码, 创建自定义的RouteLocator, 从数据库或其他数据源中读取路由信息,并动态生成路由. 亦可使用动态配置中心工具来实现动态路由动态配置等功能

  1. 身份校验

对请求身份初步校验, 减轻后端服务的负担. 同时, 可以根据请求上下文, 动态地应用授权规则(与授权服务交互)等等…事实上, 企业中身份校验和认证授权一般由专门的auth服务来实现.

  1. 负载均衡

支持轮询(默认), 随机, 响应时间加权, 最小连接数, 重试等等负载均衡策略.

  1. 监控和日志

在gateway中可以对API的响应时间, 成功率, 请求频率的指标进行统计, 以便问题追踪和审计.

  1. 限流

        filters:
        - name: RequestRateLimiter
          args:
            redis-rate-limiter.replenishRate: 10 # 每秒补充的令牌数
            redis-rate-limiter.burstCapacity: 20 # 令牌桶的容量
            redis-rate-limiter.key-resolver: "#{@myKeyResolver}" # 用于确定限流键的Bean

限流可以防止服务过载以保证系统的稳定性. Spring Cloud Gateway 提供了基于令牌桶算法的限流过滤器 RequestRateLimiterGatewayFilterFactory.

  1. 熔断

    filters:
    - name:    # 需要引入支持的熔断器的包 
      args:    # 引入的熔断器需要的参数

防止系统雪崩, 在后端服务不可用是提供回退方案.

  1. 重试

在服务暂时不可用时, 自动重试请求. 可以配置重试的次数和条件.

  1. 缓存

对于一些高频的响应数据, 可以在gateway中缓存响应信息, 以减轻后端负载

  1. 服务降级

gateway其实通常不直接管理服务的健康状态和降级策略, 而是依赖于 Eureka, Consul, hystrix等服务治理框架和工具来获取信息并决定如何处理请求
