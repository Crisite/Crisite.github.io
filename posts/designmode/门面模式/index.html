门面模式

星期五，三月 8 日，2024 年 | 1 分钟阅读 | 更新于 星期五，三月 8 日，2024 年

泡芙

门面（外观）模式旨在通过一个门面类（Faceade.java）提供一个统一的接口，以隐藏系统的复杂性并提供更方便的访问方式。客户端只需要与门面类交互，而不需要直接与子系统中的个各类进行交互

比较简单感觉没什么能讲的~

系统中有如下几个接口

  • 图片搜索
  • 文章搜索
  • 视频搜索
  • 网页搜索
  • …….

系统提供了多个接口，其他系统想要调用时，需要手动写入多个api，当接口变多时，代码将非常 不优雅

因此可以创建一个 Faceadce.java 同时 api 调用的请求参数 request 添加一个 type字段，根据不同 type 去调用不同的接口， 其他系统不需要关心本系统有多少个api，只需要关注自己需要选择什么类型的 type 即可
